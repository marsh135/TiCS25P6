# TiCS25P6

Cybersecurity Project: Password Generator & Secure Storage App

# Overview

This project introduces you to core cybersecurity concepts through hands-on development. You will build TWO applications:
  1. A Secure Password Generator
  2. A Password Storage App (Password Manager)

These tools simulate real-world systems used to protect sensitive information.


___


# Part 1: Secure Password Generator Requirements

Your generator must:
  * Generate secure, random passwords
  * Allow user input for:
    * Length
    * Character types (uppercase, lowercase, numbers, symbols)
    * Avoid predictable patterns

Cybersecurity Focus
  *  Explain (in comments or README):
  *  Why randomness matters
  *  What makes a password “strong”

Optional Enhancements
  * Strength rating (weak/medium/strong)
  * Exclude confusing characters


# Part 2: Password Storage App Core Requirements

Your app must:
  * Require authentication (login) before access
  * Store credentials using a dictionary (key-value pairs)
    * Example: { "gmail": "password123" }
  * Allow the user to:
    * Add a password
    * Retrieve a password
    * View stored account names (NOT passwords)

Required Features
  * Login system (username + password)
  * Menu system (looping program)
  * Add entry
  * Retrieve entry

Cybersecurity Focus

You must address at least ONE of the following: (WE WILL COVER THIS IN CLASS BEFORE YOU WORK ON THIS PART!)
  * Hashing the master password
  * Basic encryption of stored passwords
  * Preventing plain-text exposure



# GitHub Submission Requirements

Your repository must include:
  *  All source code
  *  A README.md that includes:
     * Project description
     * How to run the program
     * Security features implemented
     * What you would improve with more time


# Grading Checklist

Functionality (40 pts)
  *  Password generator works
  *  Storage system works
  *  Authentication works

Cybersecurity Understanding (25 pts)
  *  Demonstrates secure thinking
  *  Explains security choices in README


GitHub + Documentation (10 pts)
  *  Repo submitted
  *  README included


Extension Ideas (Optional)
  * Encrypt stored passwords
  * Multiple user accounts
  * Save/load from file (JSON)
  * GUI (Tkinter)
  * Password breach checker (conceptual)



Notes
  *  Start simple, then improve security
  *  Real-world systems are much more complex—this is a simulation
  *  Focus on understanding WHY security matters, not just making it work

Good luck—think like a hacker AND a defender.
